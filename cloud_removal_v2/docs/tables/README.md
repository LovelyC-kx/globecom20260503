# cloud_removal_v2/docs/tables/

Camera-ready tables for the v2 paper §VI. Each table ships in **two
synchronized files**:

* `table_X.md`  — canonical source (data + provenance + observations).
                  Use this when reading / reviewing / sanity-checking.
* `table_X.tex` — paper-ready LaTeX with booktabs/multirow. Drop into
                  the IEEE class file as-is.

If they ever disagree, **the .md is authoritative**. The .tex is a
literal LaTeX rendering of the same numbers.

## Index

|     File          | Paper label   | Content | Source |

## Conventions

### Numerical formatting

|   Quantity   | Stored | Displayed |
|:-------------|:------:|:---------:|
| PSNR (dB)    | float64 | 3 decimals (e.g. 21.642) |
| SSIM         | float64 | 4 decimals (e.g. 0.6636) |
| Comm (MB)    | bytes   | rounded to integer MB (3694, 5911) |
| Wall (h)     | seconds | 2 decimals (5.87) |
| Var(γ), Var(β) | float64 | scientific notation, 3 sig-fig (e.g. 7.04e-05) |
| max\|γ−1\|, max\|β\| | float64 | 4 decimals |

### Bolding rules

* Tables I & II — bold **best** within each (Run, BN) sub-block.
* Table III — bold **the FedAvg+AllReduce 0.000e+00** entries (sanity
  check) and **the largest Var per run** (always FedBN+RelaySum).
* Table IV — bold whichever value is "better" per metric (higher PSNR,
  lower wall, etc.). Ties are not bolded.

### Sub-block delimiters

LaTeX tables use `\cmidrule(lr){...}` to separate FedAvg vs FedBN
within a run, and `\midrule` to separate runs. Markdown tables rely
on the leading `Run` column for visual grouping.

## Configuration parity guarantees

All v2 tables compare runs that share these hyperparameters
(verified by comparing `summary.json["config"]` dicts byte-for-byte
after removing the deliberately different field):

```
seed=1234, partition_seed=0, partition_mode=dirichlet_source, partition_alpha=0.1
num_planes=5, sats_per_plane=10
T=4, vlif_dim=24, en_blocks=[2,2,4,4], de_blocks=[2,2,2,2]
patch_size=64, train_batch_size=4, augment=True
lr=1e-3, min_lr=1e-7, warmup_epochs=3 (cosine), num_epoch=80
intra_plane_iters=2, local_iters=2
loss = Charbonnier(eps=1e-3) + 0.1 · (1 − SSIM)
eval_mode=center_patch, eval_patch_size=64, eval_every=5
```

Comparison-clean pairs (used in the paper):

* **A vs B** — only `bn_variant` differs (`tdbn` ↔ `bn2d`).
  → BN-variant ablation. Used in Tables I, II, III.
* **A vs C** — only `backbone` differs (`snn` ↔ `ann`).
  → Backbone ablation. Used in Tables I, IV.

Comparison-dirty pair (avoided in the paper):

* **B vs C** — *two* fields differ (`bn_variant` AND `backbone`).
  → No paper claim references this pair directly.

## What is *not* in these tables (and why)

* **Error bars.** Single-seed runs do not yield error bars. Multi-seed
  validation (V9) is deferred to v3 — see `v2_remaining_issues.md`.
* **Per-epoch convergence curves.** Those are the PDFs generated by
  `plot_results.py`; they are figures, not tables. Reproduce locally.
* **Per-image PSNR distribution.** Stored in each cell's
  `*.npz["per_plane_psnr"]` arrays; not summarised here. Useful for a
  v3 multi-seed reproduction.
* **Energy.** No spike-rate measurement script has been run yet.
  §VI-E will report energy as "not measured" until the script is in.
* **Centralised upper bound.** A non-FL VLIFNet baseline would be
  useful as a ceiling. Deferred to v3 (V8 in `v2_remaining_issues.md`).
